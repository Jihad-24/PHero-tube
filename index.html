Question 1 : Discuss the scope of var, let, and const ?

Answer : var, let, and const are three different ways to declare variables in JavaScript, and they have scopes and behaviors. These keywords play a crucial role in how variables are treated.

1. var : Variables declared with var are function-scoped or globally-scoped, when it declared inside function it is only accessible within that function's scope. But if it's diclared outside of any function, it becomes a global variable, accessible throughout the entire script. var declarations are hoisted,which can sometimes lead to unexpected behavior.

2. let : Variables declared with let are block-scoped, which means they are confined to the block (curly braces{}) in which they are defined.

3. const : Variables declared with const are also block-scoped like let.The difference is that when const variables are diclared their values cannot be reassigned once they are assigned.

Question 2 : Tell us the use cases of null and undefined ?

Answer : In programming, null and undefined are two distinct concepts used to represent the absence or unavailability of a value.

1. null : Developers can assign null to a variable or property when they want to indicate that there is no meaningful value or that the value has been intentionally removed. With null we can indicate that they have not been assigned a valid value yet. Functions may return null to signify that they couldn't produce a valid result or that an error occurred.

2. undefined : In JavaScript, undefined typically represents the absence of an assigned value. If a function parameter is not provided when the function is called, it's automatically set to undefined.Developers often use undefined to check if a variable or object property exists or has been assigned a value.

Question 3 : What do you mean by REST API ?

Answer : REST, which stands for Representational State Transfer, is an architectural style and set of constraints for designing networked applications. A REST API (Application Programming Interface) is a type of web service that acording to the principles and constraints of REST. REST APIs are designed to allow different software systems to communicate with each other over the internet using standard HTTP methods.
